# System Limitations

## Overview

While the proposed deterministic voice-based POS system demonstrates strong performance in controlled environments, several limitations exist due to design constraints and research scope boundaries.

---

## Deterministic NLU Scalability

The system uses rule-based intent detection and gazetteer-based slot extraction.  
As the number of supported products increases significantly, lexical ambiguity also increases.

This can lead to:

- Reduced slot extraction accuracy  
- Confusion between phonetically similar product names  
- Increased need for alias tuning  

---

## Single Language Dependency

The current system operates using English voice transcripts generated by browser-based speech recognition.

Limitations include:

- No support for multilingual commands  
- No regional pronunciation adaptation  
- No dialect handling  

This restricts usability in diverse retail environments.

---

## Dataset Size Constraints

Evaluation experiments were conducted on controlled datasets with limited command variations.

Therefore:

- Statistical generalization is limited  
- Real-world ASR variability is not fully captured  
- Edge-case command behavior may remain untested  

---

## Lack of Contextual Dialogue Support

The system processes each command independently.

It does not support:

- conversational context  
- follow-up clarifications  
- multi-turn billing instructions  

This may limit natural interaction flow.

---

## Hardware and Environment Dependency

Voice recognition performance depends on:

- microphone quality  
- browser speech recognition accuracy  
- background noise conditions  

Thus system performance may vary across retail setups.

---

## Absence of Cloud Synchronization

The current implementation is designed for single-store local deployment.

Limitations include:

- No multi-terminal synchronization  
- No centralized inventory management  
- No remote analytics aggregation  

---

## Manual Evaluation Requirement

Some performance measurements require manually curated datasets.

Automated large-scale benchmarking pipelines are not yet implemented.

---

## Summary

These limitations provide opportunities for future research and system evolution toward scalable intelligent retail automation platforms.
